# Пункт checklist: architecture_artifact_updates

Финальная синхронизация всех агрегирующих артефактов по данным текущего репозитория перед его закрытием.

## Назначение

Этот пункт — не новый анализ, а **обязательная проверка-чеклист**: всё, что было собрано в пунктах 1–16, должно отразиться в соответствующих артефактах. Пункт нельзя считать выполненным, пока не проверен каждый из нижеперечисленных пунктов.

## Чеклист артефактов

Пройди каждый пункт и поставь галочку (или зафикируй расхождение):

### landscape.yaml

- [ ] Запись для сервиса существует
- [ ] Поле `technology` заполнено в формате `<runtime> / <framework>`
- [ ] Поле `head_commit` (или `analyzed_commit`) зафиксировано
- [ ] Зависимости (`dependencies`) отражают найденные интеграции

### tech-stack.md

- [ ] Содержит строки для сервиса по ключевым зависимостям
- [ ] Указан сервис, версия и источник подтверждения

### hld.md

- [ ] Раздел "Обзор архитектуры" отражает роль сервиса
- [ ] Секция компонента описывает реализацию (а не black-box)
- [ ] Раздел "Основные потоки" содержит новые потоки из этого репозитория

### architecture/contracts/

- [ ] `<service>-sync.yml` существует **или** notes содержит "нет sync контракта"
- [ ] `<service>-async.yml` существует **или** notes содержит "нет async контракта"

### architecture/storage/

- [ ] `<service>.yml` существует **или** notes содержит "нет хранилища"

### architecture/integrations/

- [ ] `<service>.md` существует (создан в пункте 7)

### architecture/integrations-overview.md

- [ ] Содержит строки для сервиса в разделе "Карта интеграций" (входящие + исходящие)
- [ ] Раздел "Разрез по направлению" обновлён

### architecture/domain-entities.md

- [ ] Обновлён записями из текущего репозитория **или** notes пункта `domain_entities` содержит "нет новых сущностей"
- [ ] Поля с бэкенда, не выгружаемые в API/UI, помечены как `backend_only`

### architecture/security.md

- [ ] Обновлён **или** notes пункта 14 содержит "не удалось восстановить"

### architecture/risks.md

- [ ] Обновлён **или** notes пункта 16 содержит "рисков не выявлено"

### features/

- [ ] Затронутые фичи обновлены с трассировкой реализации (точки входа, тесты, статус)

### AGENTS.md

- [ ] Сервис есть в списке текущих сервисов

### CLAUDE.md

- [ ] Существует в корне arch-репозитория и содержит только `@AGENTS.md`

## Шаги выполнения

1. Пройти чеклист выше сверху вниз.
2. Для каждого незакрытого пункта — внести исправления в соответствующий артефакт.
3. Зафиксировать в notes: что обновлено, что осталось с явной пометкой "нет / не удалось восстановить".

## Проверка согласованности

Пункт нельзя завершить (`completed`) если:

- `landscape.yaml` обновлён, а `integrations-overview.md` — нет
- Артефакт `integrations/<service>.md` создан, а в `integrations-overview.md` сервис не упомянут
- Есть контракт в `contracts/`, но в `hld.md` интерфейс не описан

## Подводные камни

- Если артефакт должен быть переиспользуемым, оформляй его по шаблону, а не свободным описанием.
- Результат не готов, если неясно, какие утверждения подтверждены, а какие предположены: каждый вывод должен иметь источник или явную пометку "не удалось восстановить".
- Результат не готов, если HLD не отражает реальный состав компонентов и потоков из проанализированных репозиториев.
- Не заполняй требования и ограничения догадками, если в репозитории нет достаточных свидетельств: лучше явная пометка "не удалось восстановить", чем непроверенное предположение.
