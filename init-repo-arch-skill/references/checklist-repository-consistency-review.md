# Пункт checklist: repository_consistency_review

Финальная проверка полноты и согласованности всего, что было собрано по данному репозиторию, перед его закрытием через `repo --complete`.

## Назначение

Этот пункт — барьер качества перед закрытием репозитория. Задача не найти новые факты, а убедиться, что ничего не потеряно, артефакты не противоречат друг другу и всё что найдено — отражено корректно.

## Проверка полноты checklist

Пройди все предыдущие пункты `analysis_checklist` и убедись, что у каждого:
- [ ] статус `completed`
- [ ] notes содержат либо findings, либо явную пометку (`"не найдено"`, `"нет хранилища"`, `"нет async контракта"` и т.п.)

Нельзя закрыть репозиторий, если хотя бы один пункт имеет статус `not_started` или пустой notes.

## Проверка внутренней согласованности артефактов

Для каждой пары — убедиться в отсутствии противоречий:

**Entrypoints ↔ Contracts**
- [ ] Каждый HTTP/gRPC endpoint из пункта 2 покрыт записью в `contracts/<service>-sync.yml`
- [ ] Каждый consumer/producer из пункта 2 покрыт записью в `contracts/<service>-async.yml`

**Contracts ↔ Integration**
- [ ] Топики из async-контракта совпадают с топиками в `integrations/<service>.md`
- [ ] Исходящие вызовы из пункта 7 совпадают с исходящими в `integrations/<service>.md`

**Storage ↔ HLD**
- [ ] Хранилища из `storage/<service>.yml` упомянуты в соответствующем компоненте `hld.md`

**Features ↔ Entrypoints**
- [ ] Для каждой фичи со статусом `implemented` есть трассировка с реальными путями из этого репозитория

**Landscape ↔ Integrations Overview**
- [ ] Зависимости в `landscape.yaml` согласуются с картой интеграций в `integrations-overview.md`

**Security ↔ Configs**
- [ ] Env-переменные с секретами из пункта 4 отражены в `security.md` (secrets management)

### Domain Entities ↔ Contracts + Storage

- [ ] Каждая сущность из `domain-entities.md` ссылается хотя бы на один источник в `contracts/` или `storage/`
- [ ] Поля, помеченные `backend_only`, не присутствуют в API-контракте (или расхождение явно задокументировано)

## Проверка открытых вопросов

- [ ] Все вопросы, которые можно было закрыть по данным этого репозитория, имеют статус `resolved`
- [ ] Вопросы с `ask_user=true` и статусом `open` вынесены пользователю

## Итоговые действия

1. Исправить все найденные расхождения.
2. Зафиксировать итог в notes: `"согласованность проверена, расхождений не найдено"` или список того, что осталось с явной пометкой неопределённости.
3. Только после этого запустить `repo --complete --name <repo>`.
