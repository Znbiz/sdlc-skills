# Как обновлять существующий архитектурный репозиторий

Используй этот workflow, когда архитектурный репозиторий уже существует и нужно подтянуть его к более свежему состоянию системы.

## Шаг 1. Зафиксируй предыдущую точку анализа

Сначала вытащи из существующих артефактов:

- какие репозитории были в scope
- какие коммиты были базой анализа
- какие артефакты считались завершенными
- какие вопросы уже были закрыты
- какие пробелы были оставлены открытыми

## Шаг 2. Обнови baseline репозиториев

Для каждого репозитория:

- переключись в основную ветку
- обнови её до актуального коммита
- зафиксируй новый `head_commit`
- сравни его с предыдущим baseline

## Шаг 3. Определи дельту

Выдели:

- репозитории без изменений
- репозитории с изменениями
- новые репозитории, которых не было в прошлом анализе
- отдельные test-repo, которые тоже изменились

## Шаг 4. Обнови только затронутые артефакты

По каждому изменившемуся репозиторию определи, затрагивает ли он:

- `features/`
- `architecture/hld.md`
- `architecture/landscape.yaml`
- `architecture/integrations/`
- `architecture/contracts/`
- `architecture/storage/`

## Шаг 5. Проведи валидацию

Проверь:

- не остались ли старые коммиты в описании после обновления
- не расходится ли обновленная архитектурная картина с фактическим кодом
- не сломалась ли связность между `features/` и `architecture/`
- не появились ли новые неподтвержденные предположения

## Шаг 6. Обнови файл прогресса

В конце обязательно зафиксируй:

- новый baseline
- обновленные артефакты
- оставшиеся вопросы
- следующий рекомендуемый шаг
